## Anti-Patterns Verdict

**LLM assessment:** Does NOT read as AI-generated. The dark monochrome terminal-launcher aesthetic is committed and on-brand (Quiet Cockpit); the tool recedes the way a pro launcher should. It passes the product slop test — a user fluent in Raycast/Warp would trust it. The one tell: two uppercase letter-tracked eyebrows ("CONNECTIONS", "SSH KEYS") are the textbook AI section-kicker pattern. In a two-section launcher that's borderline-acceptable product usage, but the styling is generic.

**Deterministic scan:** `detect.mjs` on the launcher markup (views.ts) is effectively clean — 1 hit (a 5px radius on a *settings* glyph, not the launcher). The styles.css font hits are intentional, domain-appropriate terminal fonts. No real slop in the launcher.

## Overall Impression

A confident, restrained launcher that mostly disappears into the task — exactly right for the Quiet Cockpit. Two things hold it back: the **first-run experience is cold** (a new user lands in a near-empty black void with one muted affordance), and the **list has no scaling story** (no search/filter/quick-launch). The single biggest opportunity is the empty state — it's the first thing every new user sees and it currently teaches nothing.

## What's Working

1. **Transport pills (SSH / ET / MOSH).** Color + text, instantly scannable, on-brand, never color-only. The strongest single element.
2. **Restraint as identity.** Genuinely minimal dark chrome; the launcher recedes so the terminal can be the loud part. The brand mark + wordmark now anchor it.
3. **Recognition density without clutter.** Name + target + last-connected time per row, hover edit/delete, right-click menus, an "Active sessions" group for resumable ET — a lot of recall support packed cleanly.

## Priority Issues

- **[P1] Cold first-run / empty state.** With no saved connections the launcher is the brand mark, two faint eyebrows, and a single muted "+ New connection". It doesn't read as a primary action, gives no value framing, and never explains SSH vs ET vs Mosh. A new user has nothing to orient on.
  - **Fix:** Design a real first-run: a confident "Connect to your first server" primary affordance, one line on what the app does, and the transports it speaks. Teach the interface, don't show a void.
  - **Suggested command:** /impeccable onboard

- **[P1] No search / filter / quick-launch.** The connection list is a flat column with no way to filter or type-to-launch. Fine at 4 hosts, painful at 40. Power users expect type-to-filter or a command palette as the default path.
  - **Fix:** Add a filter/quick-launch input (or ⌘K palette) that narrows by name/host/user and launches on Enter.
  - **Suggested command:** /impeccable shape (then craft)

- **[P2] Composition wastes the canvas.** At desktop widths the content sits top-left in a large empty void; the right rail (Settings + SSH keys) is under-weighted and disconnected. The layout doesn't compose the space — it just leaves it black.
  - **Fix:** Rebalance: tighten the column relationship, give the page a deliberate center of gravity, or treat the right rail as a quieter utility footer.
  - **Suggested command:** /impeccable layout

- **[P2] Secondary text fails contrast.** Connection meta (last-connected time, hints) uses --faint (#6b7079) on the #16171b body ≈ 3.6:1 — below WCAG AA 4.5:1 for this 12px text. (Eyebrows were already bumped to --label and now pass.)
  - **Fix:** Lift --faint toward the --label/--muted end for any text role, keep the darkest faint for non-text hairlines only.
  - **Suggested command:** /impeccable audit

- **[P2] Generic eyebrow cadence.** The uppercase tracked "CONNECTIONS" / "SSH KEYS" labels are the AI section-kicker tell. Two on one screen is borderline; the treatment is generic.
  - **Fix:** Pick a different cadence — a quieter sentence-case label, an inline count, or drop the label where the content is self-evident.
  - **Suggested command:** /impeccable typeset

## Persona Red Flags

**Alex (Power User):** No type-to-filter and no command palette — must mouse to a row to launch. Right-click menus and Enter/Space activation exist but aren't discoverable. Launch itself is fast, but the list doesn't scale to a power user's host count.

**Jordan (First-Timer):** Lands on a near-empty black screen. The only forward affordance ("+ New connection") is muted and easy to miss. No explanation of what to put in "address", or what SSH/ET/Mosh mean. The inline "Keys are added when you create a connection" is a nice touch but it's the only guidance anywhere.

**Sam (Accessibility):** Rows are keyboard-activable (role=button, Enter/Space) and the focus ring is now visible — good. Transport state is color + text (not color-only) — good. But --faint meta text at ~3.6:1 fails AA, and would be hard to read at low vision.

## Minor Observations

- Delete is confirm-only with no undo; fine for a launcher, but a toast-with-undo would be friendlier.
- The "Active sessions" (resumable ET) group is a strong idea that's easy to miss when empty — it only appears when sessions exist, which is correct, but new users never learn it's there.
- Right-click is a first-class interaction here but has zero discoverability cue.

## Questions to Consider

- What would a confident first-run look like — a single "Connect to your first server" CTA with the three transports named beneath it?
- As the saved-host list grows, should type-to-filter (or ⌘K) become the primary launch path rather than mouse-to-row?
- Does the right rail (Settings + keys) deserve equal column weight, or should it be a quieter footer so the connections own the page?
